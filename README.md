# Inventory (سیستم انبارداری تحت وب)

یک صفحهٔ HTML ساده برای ثبت و مدیریت موجودی انبار روی مرورگر. داده‌ها در LocalStorage ذخیره می‌شود و بدون نیاز به پایگاه داده یا بک‌اند کار می‌کند.

## نحوهٔ اجرا
1. مخزن را کلون کنید:
   ```bash
   git clone <YOUR_REPO_URL>.git
   cd inventory
   ```
2. یک وب‌سرور ساده اجرا کنید (مثلاً با پایتون):
   ```bash
   python3 -m http.server 8000
   ```
3. در مرورگر به آدرس `http://localhost:8000` بروید و فایل `index.html` را باز کنید.

### اجرا از طریق GitHub (GitHub Pages)
اگر می‌خواهید پروژه مستقیماً از خود GitHub اجرا شود و لینک قابل اشتراک داشته باشید:
1. همهٔ فایل‌ها (به‌ویژه `index.html`) را کامیت و به شاخهٔ اصلی (`main` یا `master`) پوش کنید.
2. وارد مخزن در GitHub شوید و از تب **Settings** بخش **Pages** را باز کنید.
3. در بخش **Build and deployment** گزینهٔ **Source** را روی «Deploy from a branch» بگذارید.
4. شاخهٔ اصلی و پوشهٔ `/root` را انتخاب کنید و ذخیره کنید.
5. چند ثانیه صبر کنید تا GitHub Pages فعال شود. آدرس عمومی در همان صفحه نمایش داده می‌شود (معمولاً `https://<USERNAME>.github.io/<REPO>/`).
6. برای به‌روزرسانی، کافی است تغییرات جدید را پوش کنید؛ چند لحظه بعد در صفحهٔ GitHub Pages منعکس می‌شود.

### انتشار تغییرات روی GitHub
اگر در GitHub تغییری مشاهده نمی‌کنید، احتمالاً هنوز ریموت اضافه نشده یا `git push` انجام نشده است. مراحل زیر را دنبال کنید:
1. ریموت را اضافه کنید (یک بار کافی است):
   ```bash
   git remote add origin git@github.com:<YOUR_USERNAME>/<REPO>.git
   ```
2. مطمئن شوید روی شاخهٔ صحیح هستید (مثلاً `main` یا شاخهٔ کاری موردنظر):
   ```bash
   git branch --show-current
   ```
3. کامیت‌های محلی را به GitHub بفرستید:
   ```bash
   git push -u origin <BRANCH_NAME>
   ```
4. در صورت داشتن SSH key، مطمئن شوید کلید عمومی در GitHub ثبت شده و Agent در سرور فعال است (`ssh-add -l`).
5. بعد از `push`، صفحهٔ مخزن را در GitHub رفرش کنید تا فایل‌های به‌روز را ببینید.

## نکات استقرار روی سرور
- در سرور هتزنر یا هر VPS دیگر می‌توانید همین مخزن را با SSH کلون کرده و وب‌سرور ساده بالا بیاورید.
- اگر وب‌سرور Nginx یا Apache دارید، کافی است پوشهٔ مخزن را به‌عنوان روت استاتیک سرو کنید تا فایل `index.html` در آدرس اصلی بارگذاری شود.
- برای به‌روزرسانی نسخه جدید، فقط `git pull` کنید و سرویس وب استاتیک را ری‌لود نمایید.
